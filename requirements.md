# 두과자

- 모든 구현은 PROJECT_STRUCTURE.md 파일을 기준으로 진행되어야한다.
- 프론트 구현 시 @/lib/axios/request.ts에 정의된 API만 우선 사용한다.
    - 필요한 API가 없으면 임의로 추가 구현하지 말고, 부족한 엔드포인트 목록과 사용 목적을 먼저 보고한다.
    - 사용자 승인 없이 API 스펙, 엔드포인트, 요청/응답 구조를 임의 생성하지 않는다.
- UI 기준은 데스크탑 화면으로, 한국어 사용자를 대상으로 한다.

## 메인 기능

- **인증/인가**
    - 사용자 인증/인가
    - 비밀번호는 해시한 값을 저장
- **과자 추천**
    - 먹고싶은 과자를 추천
    - 다른 사용자가 댓글을 달 수 있음
- **과자 피드백**
    - 과자의 상태를 설정 가능
    - 좋아요/싫어요로 피드백 가능
    - 인당 좋아요/싫어요 중 하나만 가능
- **제외 기능**
    - 관리자 페이지 구현은 MVP 제외
    - 구매한 과자 CRUD UI는 MVP 제외
    - 알림 기능 제외
    - 검색/필터 제외
    - 이미지 업로드 제외
    - 대댓글 제외
    - 신고 기능 제외
    - 소셜 로그인 제외

- 권한 정보
    일반 사용자
        - 구매한 과자 조회 가능
        - 구매한 과자에 댓글 작성 가능
        - 구매한 과자에 좋아요/싫어요 가능
        - 과자 추천글 생성/수정/삭제 가능
        - 다른 사용자의 과자 추천글 조회 가능
        - 자신의 추천글/ 다른 사용자의 추천글 댓글 작성 가능
        - 자신의 추천글/ 다른 사용자의 추천글 좋아요/싫어요 가능
    관리자
        - 구매한 과자 CRUD 가능
        - 회원 관리 가능

- 기능 설명
    - 구매한 과자
        - 관리자가 관리하는 목록.
        - 과자의 상태는 모두가 변경할 수 있다. (배송중|재고있음|재고없음)
        - 관리자는 생성, 수정, 삭제가 가능하다.
        - 일반 사용자는 댓글, 좋아요, 싫어요, 상태변경이 가능하다.
        - 댓글 기능과 피드백 기능이 존재한다.
    - 과자 추천
    - 댓글
        - 대댓글이 없는 1단계 구조
        - 댓글 작성자만 수정/삭제가 가능하다.
        - 오래된 댓글이 위에 오도록 정렬한다.
        - 5개가 넘어가는 경우 펼치기/접기가 가능해야한다.
            - 디폴트는 접힌 상태이다.
        - 댓글 기능과 피드백 기능이 존재한다.
    - 피드백
        - 좋아요/싫어요 기능
        - 사용자 당 좋아요 또는 싫어요 하나만 적용할 수 있다.

- 반에서 공용 간식을 구매할 때, 반 구성원들의 취향을 공유하기 위한 서비스
    - 구매한 과자 : 어드민이 관리한다.
        - 반장 등 관리자가 구매한 과자 리스트를 생성/수정/삭제 가능 
            (admin 페이지로 넘길 것이기 때문에, 일단은 MVP에서 제외)
        - 다른 구성원은 구매한 과자에 대해 조회, 댓글, 피드백만 가능하다. (피드백은 좋아요/싫어요를 의미한다.)
    - 과자 추천
        - 반 구성원이 먹고싶은 과자를 추천한다.
        - 생성/수정/삭제/조회가 자유롭게 가능하다.
        - 댓글과 피드백도 가능하다.(피드백은 좋아요/싫어요를 의미한다.)

필요 페이지
1. 로그인
2. 회원가입
3. 유저 정보 수정
    - nickname, password 수정이 가능하다.
    - 회원 탈퇴가 가능하다.
4. 메인 페이지
    - 아래 요소를 한 화면에 보여준다.
    - 10개씩 pagenation이 적용된다.
    - 구매한 과자 리스트
        - 댓글과 피드백(피드백은 좋아요/싫어요를 의미한다.)이 바로 보여야한다.
        - 댓글이 5개가 넘어갈 시 펼치기/접기가 가능하다. (default: 접기)
    - 과자 추천 리스트
        - 댓글과 피드백(피드백은 좋아요/싫어요를 의미한다.)이 바로 보여야한다.
        - 댓글이 5개가 넘어갈 시 펼치기/접기가 가능하다. (default: 접기)
